## 1. 리스크 계층 구조 (우선순위 순)

```
[1] 레짐 게이트       STRESSED → 전체 차단
[2] 일일 손실 게이트   PnL ≤ -1.5% → 신규 진입 차단
[3] 총 노출 하드캡     ≥ 1.05x → 전체 차단
[4] 총 노출 소프트캡   ≥ 0.90x → 버짓 0.18 상한
[5] 손실 압력 감산    손실 많을수록 버짓 순차 감소
[6] 데스크 블록       데스크별 손실 임계 초과 시 차단
[7] 종목 블록         종목별 반복 손실 시 쿨다운
[8] 스탑 압력 스케일링 최근 스탑아웃 빈도로 사이즈 축소
[9] 포지션 캡         데스크당 최대 3포지션
[10] 라이브 보수 모드  미체결 주문 존재 시 버짓 축소
```

## 3. 데스크 진입 차단 규칙

### 데스크 손실 압력
| 레벨 | 크립토 조건 | 한국주식 조건 | 효과 |
|------|-----------|-------------|------|
| medium | 손실 ≥ 2회 또는 PnL ≤ -3.0% | 동일 | 사이즈 ×0.5 |
| high | 손실 ≥ 3회 또는 PnL ≤ -4.0% | 손실 ≥ 3회 또는 PnL ≤ -5.0% | 신규 진입 차단 |

### 데스크 만성 손실 하드락
| 데스크 | 조건 | 효과 |
|--------|------|------|
| 크립토 | 승 ≤ 1 AND 패 ≥ 4 AND PnL ≤ -6.0% | 하드락 |
| 미국 | 승 = 0 AND 패 ≥ 4 AND PnL ≤ -6.0% | 하드락 |
| 한국 | 승 ≤ 1 AND 패 ≥ 4 AND (PnL ≤ -7.5% OR 스탑 ≥ 3회) | 하드락 |

### 종목별 쿨다운
| 조건 | 효과 |
|------|------|
| 최근 4거래 중 손실 발생 | 해당 종목 신규 진입 차단 |
| 최근 8거래 중 손실 2회↑ | 해당 종목 차단 |
| 최근 5거래 스탑 2회↑ 또는 누적 -5.0% (한국) | 연장 차단 |

## 4. 사이즈 계산 전체 과정

```
① base_notional = 계획 사이즈
   probe_longs:    1.0x (BALANCED) / 1.3x (OFFENSE)
   selective_probe: 0.70x
   balanced pilot:  0.60x

② desk_offense_multiplier
   오펜스 스코어 ≥ 67 → ×1.1 (press)
   오펜스 스코어 52~66 → ×1.0 (balanced)
   오펜스 스코어 < 52  → ×0.75 (cooldown)

③ symbol_edge_multiplier
   엣지 스코어 ≥ 0.7  → ×1.08 (hot)
   엣지 스코어 ≤ -0.9 → ×0.70 (cold, 진입 차단)
   엣지 스코어 ≤ -0.35 → ×0.82 (cool)
   그 외              → ×1.00 (neutral)

④ × risk_budget (0.18 ~ 0.70)

⑤ × stop_pressure_scale
   medium 압력 → ×0.5
   high 압력   → 진입 차단

최종 사이즈 = ① × ② × ③ × ④ × ⑤
```

**예시 (1천만원, BALANCED, 리스크버짓 0.5):**
```
probe_longs 1.0x × 1.0(오펜스) × 1.0(엣지) × 0.5(버짓) = 0.50x
= 5,000,000원 포지션 (주문 최대 2,500,000원 제한 적용)
```

## 5. 데스크 오펜스 스코어 계산

```python
기준점 = 50.0
score += clamp(realized_pnl_pct × 7.5, -22.0, +18.0)  # 실현손익
score += clamp((win_rate - 50.0) × 0.35, -14.0, +12.0)  # 승률
score += min(closed_count × 1.8, 8.0)                    # 거래볼륨
score += (desk_multiplier - 1.0) × 50.0                  # 자본 오버레이
score -= max(open_notional - 0.55, 0.0) × 18.0           # 과노출 패널티
score = clamp(0, 100)
```

| 구간 | 모드 | 사이즈 배수 |
|------|------|-----------|
| ≥ 67 | press | ×1.1 |
| 52~66 | balanced | ×1.0 |
| < 52 | cooldown | ×0.75 |

## 8. 라이브 실행 안전장치

### Upbit 파일럿 가드
```
UPBIT_PILOT_MAX_KRW=2500000       # 주문당 최대 250만원
UPBIT_PILOT_SINGLE_ORDER_ONLY=false  # 멀티 포지션 허용
```

### 중복 주문 방지
- (데스크, 종목, 진입/청산) 조합이 이미 pending이면 차단

### 라이브 오더 상태 추적
```
pending              → 체결 대기
awaiting_balance_sync → 잔고 동기화 대기
partial_balance_sync  → 부분 동기화
linked_partial_open   → 부분 체결 오픈
partial_close_pending → 부분 청산 대기
```
