> 최종 업데이트: 2026-04-25  
> 운용 자산: 업비트(크립토) · KIS(한국주식) · 미국주식(리서치)

## 1. 시스템 개요

### 구조
- **멀티 데스크 자율 트레이딩 시스템**
- 15분 사이클로 전체 에이전트를 병렬 실행
- 각 데스크(Crypto / Korea / US)가 독립적으로 신호 생성 → 공통 리스크 위원회가 최종 승인
- 실행 모드: `paper` / `upbit_live` / `kis_live`

### 운용 자본 구조
| 항목 | 설정값 |
|------|--------|
| 페이퍼 자본 | 10,000,000 KRW |
| 실전 자본 | `.env`의 `LIVE_CAPITAL_KRW` |
| 최대 총 노출 (hard cap) | 자본의 1.05x |
| 기본 리스크 버짓 | 0.4~0.5 |

## 2. 벤치마킹 트레이더 & 철학

시스템에 하드코딩된 4가지 트레이더 원칙:

| 트레이더 | 원칙 | 반영 방식 |
|---------|------|-----------|
| **Paul Tudor Jones** | "defense first" | DEFENSE 스탠스 시 리스크버짓 0.3, 신규 진입 최우선 차단 |
| **Stan Druckenmiller** | "size with conviction" | 데스크 오펜스 스코어 높을수록 사이즈 곱수(최대 1.1x) 증가 |
| **Linda Raschke** | "price confirms" | TrendStructureAgent가 가격 확인 후 bias 결정 |
| **Ed Seykota** | "systems and risk over prediction" | RiskCommitteeAgent가 모든 주관적 판단 배제 후 수치 임계값으로만 판단 |

## 3. 매매 전략 — 공통 원칙

### 브레이크아웃 스윙 전략 (전 데스크 공통)

**진입 조건 (4가지 모두 또는 3가지 이상 충족):**
1. **N기간 신고가 돌파** — 직전 N봉 최고가 초과
2. **거래량 서지** — 현재 거래량 ≥ N기간 평균의 3.0x (크립토) / 2.5x (한국주식)
3. **RSI 모멘텀 구간** — RSI 50~72 (크립토) / 55~78 (한국주식)
4. **EMA 위 위치** — 종가 > EMA(20)

**브레이크아웃 스코어:**
| 충족 조건 수 | 스코어 | 상태 |
|------------|--------|------|
| 4/4 | 0.90 | `breakout_confirmed` |
| 3/4 | 0.70 | `breakout_partial` |
| 2/4 | 0.45 | 약한 신호 |
| 1/4 | 0.20 | 무시 |

**손익 목표 (전 데스크 공통 기준):**
- 목표가: **+4.0%**
- 손절가: **-2.0% (크립토) / -2.5% (한국주식)**
- 리스크/리워드: 2.0:1

|------|------|------|------|
| KRW-DOGE | 51.81% | ~53% | 5.47 |
| KRW-XRP | 48.19% | ~53% | 5.47 |

- 방향성 기준봉: **KRW-BTC** (방향 필터)
- 15분봉 기준, 40봉 룩백

### 신호 계산 (summarize_crypto_signal)
```
기본 점수 = 0.5
+0.15  EMA10 > EMA30 (골든크로스)
-0.15  EMA10 < EMA30 (데드크로스)
+0.10  RSI 45~68 구간 (균형 모멘텀)
+0.10  5봉 모멘텀 > +1.0%
-0.10  5봉 모멘텀 < -1.0%
+0.08  컨트롤드 브레이크아웃 (0.5~2.4% / 풀백갭 ≤1.6%)
-0.10  3봉 버스트 > 2.4% (과열)
+0.15  브레이크아웃 4/4 확인
+0.08  브레이크아웃 3/4 확인
+0.03  브레이크아웃 2/4 확인
```

**바이어스 분류:**
- score ≥ 0.62 → `offense`
- score ≤ 0.40 → `defense`
- 그 외 → `balanced`

### 진입 결정 트리

| 조건 | 액션 | 사이즈 |
|------|------|--------|
| 과열 (5봉 +3.4% 이상 또는 RSI≥74) | watchlist_only | 0x |
| 약세 (5봉 -2.8% 이하) | capital_preservation | 0x |
| offense + signal≥0.72 + **breakout_confirmed** | probe_longs | 0.65x (BALANCED) / 0.85x (OFFENSE) |
| offense + signal≥0.67 + **breakout_partial** | selective_probe | 0.40x |
| defense (mild) | watchlist_only | 0x |
| balanced + signal≥0.60 + **breakout_partial** | probe_longs | 0.30x |
| 기본 | watchlist_only | 0x |

**⚠️ probe_longs는 반드시 breakout_confirmed(4/4) 필요 — 미확인 진입 금지**

### 포지션 관리
- 목표: +4.0%
- 손절: -2.0%
- 최대 보유: 720사이클 (24시간)
- 조기 실패 탈출: 30사이클 후 PnL ≤ -1.4%
- 스테일 퇴출: 최대사이클 직전 PnL < +0.6%
- 동시 최대 포지션: 2개, 총 노출 1.2x

## 7. 리스크 관리 체계

### 스탠스 결정 (매 사이클)
```
매크로 점수 + 트렌드 점수 → 합산

합산 ≥ 0.66 → OFFENSE  (리스크버짓 0.7)
합산 ≤ 0.42 → DEFENSE  (리스크버짓 0.3)
그 외       → BALANCED (리스크버짓 0.5)
```

### 레짐 분류
| 레짐 | 조건 | 영향 |
|------|------|------|
| STRESSED | 매크로 점수 ≤ 0.35 | 전 데스크 신규 진입 차단 |
| RANGING | 트렌드점수 - 0.5 의 절대값 ≤ 0.08 | 기본 임계값 적용 |
| TRENDING | 그 외 | offense 임계값 소폭 완화 |

### 리스크 버짓 감산 체계 (순서대로 적용)
```
1. 레짐 STRESSED          → 전체 차단
2. 일일 PnL ≤ -1.5%       → 신규 진입 차단
3. 총 노출 ≥ 1.05x         → 전체 차단 (hard cap)
4. 총 노출 ≥ 0.90x         → 버짓 0.18 상한
5. 손실 > 수익 or PnL ≤ -0.75% → 버짓 0.2 상한
6. PnL < 0%               → 버짓 0.3 상한
```

### 데스크별 진입 차단 규칙
| 조건 | 크립토 | 한국주식 |
|------|--------|---------|
| 손실 누적 ≤ -4% (크립토) / -5% (한국) | 차단 | 차단 |
| 손실 3회 이상 | 차단 | 차단 |
| 만성 손실 (승0 패4↑ PnL≤-6%) | 하드락 | 하드락 |
| 최근 4거래 동일종목 손실 | 쿨다운 | 쿨다운 |

### 사이즈 계산 체인
```
base_notional (계획 사이즈)
  × desk_offense_multiplier  (0.75x ~ 1.1x, 데스크 성과 기반)
  × symbol_edge_multiplier   (0.70x ~ 1.08x, 종목별 승률 기반)
  × risk_budget              (0.3 ~ 0.7)
  × stop_pressure_scale      (0.5x if 중간 스탑압력)
= 최종 주문 사이즈
```

### 데스크 오펜스 스코어링
```
기준점 = 50
+max(realized × 7.5, 18) / -22  실현손익
+max((승률-50) × 0.35, 12) / -14 승률
+min(거래횟수 × 1.8, 8)          거래볼륨 보너스
-max(노출-0.55, 0) × 18          과노출 패널티

≥ 67점 → press   (사이즈 ×1.1)
52~66점 → balanced (사이즈 ×1.0)
< 52점 → cooldown (사이즈 ×0.75)
```

## 8. 진입/청산 트리거 전체 목록

### 포지션 자동 청산 조건 (우선순위 순)
1. **target_hit** — PnL ≥ +4.0%
2. **stop_hit** — PnL ≤ -2.0% (크립토) / -2.5% (한국)
3. **early_failure** — 30사이클(크립토)/20사이클(한국) 후 PnL ≤ -1.4%/-1.75%
4. **stale_exit** — 최대사이클 직전 PnL < +0.6%
5. **time_exit** — 최대사이클 초과 (크립토 720 / 한국 195)

### 복구 조건 (데스크 재활성화)
최근 2거래 누적 수익 ≥ +1.5% 이상이어야 쿨다운 해제
