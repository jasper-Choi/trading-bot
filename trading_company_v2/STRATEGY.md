## 7. 리스크 관리 체계

### 스탠스 결정
```
합산 ≥ 0.66 → OFFENSE  (리스크버짓 0.7)
합산 ≤ 0.42 → DEFENSE  (리스크버짓 0.3)
그 외       → BALANCED (리스크버짓 0.5)
```

### 레짐 분류
| 레짐 | 조건 | 영향 |
|------|------|------|
| STRESSED | 매크로 점수 ≤ 0.35 | 전 데스크 신규 진입 차단 |
| RANGING | 트렌드점수 편차 ≤ 0.08 | 기본 임계값 적용 |
| TRENDING | 그 외 | offense 임계값 소폭 완화 |

### 리스크 버짓 감산 체계
```
1. 레짐 STRESSED          → 전체 차단
2. 일일 PnL ≤ -1.5%       → 신규 진입 차단
3. 총 노출 ≥ 1.05x         → 전체 차단 (hard cap)
4. 총 노출 ≥ 0.90x         → 버짓 0.18 상한
5. 손실 > 수익 or PnL ≤ -0.75% → 버짓 0.2 상한
6. PnL < 0%               → 버짓 0.3 상한
```

### 사이즈 계산 체인
```
base_notional (1.0x~1.3x)
  × desk_offense_multiplier  (0.75x ~ 1.1x)
  × symbol_edge_multiplier   (0.70x ~ 1.08x)
  × risk_budget              (0.3 ~ 0.7)
  × stop_pressure_scale      (0.5x if 중간 스탑압력)
= 최종 주문 사이즈
```
